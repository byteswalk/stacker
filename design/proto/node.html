Node 生态页

「管理器托管」+ fnm 接管。fnm 通过 shell 钩子在每个终端将当前版本目录前置到 PATH 最前，因而优先于本机现有的 nvm / 手动安装的 node——无论现有环境多复杂都能接管。代价是必须注入 shell 集成，因此顶部设有一条 shell 集成状态条（这是 fnm 版判断"是否生效"的关键，取代了 nvm 的"被覆盖"判断）。共 7 扇帧 + 弹窗。
